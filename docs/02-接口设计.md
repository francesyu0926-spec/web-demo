# 观点科技电子招投标交易平台 — 接口设计

> 风格：RESTful，前后端分离，JSON 交互。前缀统一 `/api`。
> 鉴权：除"游客可访问"接口外，均需 `Authorization: Bearer <JWT>`。
> 角色码：`GUEST / BIDDER(投标人) / AGENT(项目经理-招标代理) / EXPERT(专家) / ADMIN`。

---

## 0. 通用约定

### 统一响应体
```json
{ "code": 0, "msg": "success", "data": {} }
```
- `code=0` 成功，非 0 为业务错误码；HTTP 状态码语义同时保留（401/403/404/500）。

### 分页约定
- 入参：`pageNum`(默认1) / `pageSize`(默认10) / 其余筛选字段。
- 出参：
```json
{ "list": [], "total": 100, "pageNum": 1, "pageSize": 10 }
```

### 权限标注
每个接口标注 `🔓游客` 或所需角色，如 `[AGENT]`。写操作另需校验**项目状态机**（见总览文档）。

## 1. 认证与用户 `/api/auth` `/api/users`

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| POST | `/api/auth/register` | 🔓 | 账号/手机注册 |
| POST | `/api/auth/sms-code` | 🔓 | 发送短信验证码 |
| POST | `/api/auth/login` | 🔓 | 登录(账号密码/手机验证码/微信) |
| POST | `/api/auth/wechat/login` | 🔓 | 微信登录(code换openid) |
| POST | `/api/auth/wechat/bind` | 登录态 | 绑定/换绑手机 |
| GET | `/api/auth/me` | 登录态 | 当前用户+拥有角色+当前激活角色 |
| POST | `/api/auth/switch-role` | 登录态 | 切换激活角色，返回新 token |
| POST | `/api/auth/logout` | 登录态 | 退出 |
| GET | `/api/users/profile` | 登录态 | 个人资料 |
| PUT | `/api/users/profile` | 登录态 | 修改资料 |

**POST /api/auth/login 请求**
```json
{ "loginType": "PASSWORD|SMS|WECHAT", "account": "13800000000", "credential": "密码或验证码" }
```
**响应 data**
```json
{ "token": "xxx", "userId": 1, "roles": ["BIDDER","EXPERT"], "activeRole": "BIDDER" }
```

**POST /api/auth/switch-role 请求**
```json
{ "role": "EXPERT" }
```

## 3. 招标项目（游客查询 + 代理管理）

### 3.1 对外查询（游客）
| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/tenders` | 🔓 | 找招标(关键词/地区/行业/招标方式/状态 筛选+分页) |
| GET | `/api/tenders/{id}` | 🔓 | 招标详情 |
| GET | `/api/announcements` | 🔓 | 公告列表 `?type=1招标/2候选人公示/3结果` |
| GET | `/api/winners` | 🔓 | 找中标(中标公示列表) |

**GET /api/tenders 入参**：`keyword, region, industry, category, status, pageNum, pageSize`
**列表项 data.list[]**
```json
{ "id":1, "name":"xxx项目", "projectNo":"ZB2026001", "category":"公开招标",
  "region":"山西", "budget":1000000.00, "status":"PUBLISHED",
  "regEnd":"2026-06-20 18:00", "publishTime":"2026-06-03 10:00" }
```

### 3.2 项目管理（招标代理 = 项目经理）
| 方法 | 路径 | 权限 | 状态校验 | 说明 |
|---|---|---|---|---|
| GET | `/api/agent/tenders` | [AGENT] | - | 我管理的项目列表 |
| POST | `/api/tenders` | [AGENT] | →DRAFT | 创建项目 |
| PUT | `/api/tenders/{id}` | [AGENT] | DRAFT | 编辑 |
| POST | `/api/tenders/{id}/publish` | [AGENT] | DRAFT→PUBLISHED | 发布招标公告 |
| POST | `/api/tenders/{id}/abort` | [AGENT] | any→ABORTED | 废标/流标 `{reason}` |
| GET | `/api/tenders/{id}/progress` | [AGENT/EXPERT] | - | 查看进度(各阶段状态) |

**POST /api/tenders 请求**
```json
{ "name":"xxx", "category":"公开招标", "industry":"工程", "region":"山西",
  "budget":1000000, "depositAmount":20000,
  "regStart":"2026-06-03 09:00", "regEnd":"2026-06-20 18:00",
  "bidOpenTime":"2026-06-21 09:30", "content":"招标公告正文...",
  "evaluationItems":[ {"type":"TECH","name":"技术方案","maxScore":40,"weight":40} ] }
```

## 4. 投标（投标人 + 代理审核）

| 方法 | 路径 | 权限 | 状态校验 | 说明 |
|---|---|---|---|---|
| POST | `/api/tenders/{id}/registrations` | [BIDDER] | 项目=PUBLISHED | 投标报名 |
| GET | `/api/my/registrations` | [BIDDER] | - | 我的报名 `?bidStatus=` |
| GET | `/api/registrations/{id}` | [BIDDER/AGENT] | - | 报名/投标详情 |
| POST | `/api/registrations/{id}/documents` | [BIDDER] | 报名已通过 & 未截止 | 递交投标文件 `{attachId,bidPrice}` |
| POST | `/api/registrations/{id}/deposit` | [BIDDER] | - | 缴纳保证金(待定) |
| GET | `/api/tenders/{id}/registrations` | [AGENT] | - | 报名审核列表 |
| PUT | `/api/registrations/{id}/audit` | [AGENT] | 项目=PUBLISHED/REG_CLOSED | 报名审核 `{auditStatus:1/2, remark}` |

**我的报名列表项**
```json
{ "id":10, "projectId":1, "projectName":"xxx", "auditStatus":1,
  "bidStatus":"SUBMITTED", "bidPrice":980000.00, "regEnd":"...", "bidOpenTime":"..." }
```
> bidStatus：REGISTERED 报名/SUBMITTED 已投标/WON 已中标/LOST 未中标/INVALID 废标。

## 5. 开评标（代理 + 专家）

### 5.1 代理：抽专家 / 开标 / 定标
| 方法 | 路径 | 权限 | 状态校验 | 说明 |
|---|---|---|---|---|
| POST | `/api/tenders/{id}/experts/draw` | [AGENT] | REG_CLOSED | 抽取评标专家 `{count, majors}` 或 `{expertIds}` |
| GET | `/api/tenders/{id}/experts` | [AGENT] | - | 已抽取专家及确认状态 |
| POST | `/api/tenders/{id}/open` | [AGENT] | REG_CLOSED→OPENED | 开标(到点/手动) |
| GET | `/api/tenders/{id}/evaluation/summary` | [AGENT] | EVALUATING+ | 评标汇总(各投标人加权得分/排名) |
| POST | `/api/tenders/{id}/award` | [AGENT] | EVALUATING→WIN_PUBLICITY | 定标/发布中标公示 `{ranks:[{registrationId,rank}], winnerRegistrationId, publicityStart, publicityEnd}` |
| POST | `/api/tenders/{id}/finish` | [AGENT] | WIN_PUBLICITY→FINISHED | 结束项目 |

### 5.2 专家：我的项目 / 签到 / 评分
| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| GET | `/api/expert/assignments` | [EXPERT] | 我的项目 `?status=PENDING/ACCEPTED/SIGNED` |
| POST | `/api/assignments/{id}/respond` | [EXPERT] | 确认/拒绝 `{accept:true/false}` |
| POST | `/api/assignments/{id}/sign-in` | [EXPERT] | 签到(开标当天) |
| GET | `/api/tenders/{id}/evaluation/items` | [EXPERT] | 评审表(评审项+投标人清单) |
| GET | `/api/tenders/{id}/bids` | [EXPERT] | 开标后可下载投标文件 |
| POST | `/api/evaluations/scores` | [EXPERT] | 提交/暂存评分(批量) |

**POST /api/evaluations/scores 请求（批量）**
```json
{ "projectId": 1, "submitted": true,
  "scores": [
    { "registrationId": 10, "itemId": 100, "pass": 1 },
    { "registrationId": 10, "itemId": 200, "score": 38.5, "remark": "技术方案完整" }
  ] }
```
> 形式/资格/响应性评审用 `pass`(0/1)；商务/技术用 `score`。`submitted=false` 为暂存。

## 7. 错误码建议（节选）

| code | 含义 |
|---|---|
| 0 | 成功 |
| 1001 | 未登录/Token失效 |
| 1003 | 无权限(角色不匹配) |
| 2001 | 项目状态不允许该操作 |
| 2002 | 报名已截止 |
| 2003 | 重复报名 |
| 2004 | 开标前不可下载投标文件 |
| 3001 | 专家未签到不可评分 |
| 5000 | 系统异常 |

## 8. 包结构建议（Spring Boot）

```text
com.guandian.bidding
├─ common        统一响应/异常/分页/枚举/工具
├─ config        Security(JWT)、Knife4j(Swagger)、MyBatis-Plus、CORS
├─ security      JWT过滤器、UserDetails、@PreAuthorize 角色校验
├─ module
│  ├─ auth       认证/用户/角色切换
│  ├─ supplier   投标人资料
│  ├─ expert     专家资料/申请
│  ├─ tender     招标项目(含状态机 TenderStateMachine)
│  ├─ bid        报名/投标文件/保证金
│  ├─ evaluation 抽专家/评审项/评分/定标
│  ├─ content    公告/文章/投诉
│  └─ notify     消息通知
└─ infra         文件存储(MinIO)、短信、微信
```

> 推荐依赖：Spring Boot 3.x、Spring Security + JWT、MyBatis-Plus、MySQL、Redis(验证码/缓存)、Knife4j(接口文档)、MinIO。
