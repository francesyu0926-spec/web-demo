# 观点科技电子招投标交易平台 — 接口设计

> 依据《观点招投标web版功能清单v0603-1》及《研发需求文档 V1.4》更新。风格：RESTful + JSON，前缀 `/api`。
> 功能对照：[03-功能清单v0603](./03-功能清单v0603.md)
> 鉴权：除"🔓游客"接口外均需 `Authorization: Bearer <JWT>`。
> 角色码：GUEST / BIDDER(投标人) / TENDERER(招标人) / MANAGER(项目经理) / EXPERT(专家) / ADMIN。

---

## 0. 通用约定

- 统一响应：`{ "code": 0, "msg": "success", "data": {} }`。
- 分页入参 `pageNum/pageSize`，出参 `{ list, total, pageNum, pageSize }`。
- 每个接口标注权限角色；写接口另需校验**项目状态 + 评审节点**（见 00-总览）。

## 1. 认证 / 用户 / 角色 `/api/auth` `/api/users`

|---|---|---|---|
| POST | `/api/auth/register` | 🔓 | 注册(账号/手机/微信绑定) |
| POST | `/api/auth/sms-code` | 🔓 | 发送短信验证码 |

| POST | `/api/auth/switch-role` | 登录态 | 角色切换(身份切换),重签token |
| GET/PUT | `/api/users/profile` | 登录态 | 个人资料(含权限信息) |

| GET/PUT | `/api/users/preference` | [BIDDER] | 我的偏好(商机推荐用) |
| POST | `/api/role-applications` | 登录态 | 申请成为 项目经理/专家 |

## 10. 错误码（节选）

| code | 含义 | | code | 含义 |
|---|---|---|---|---|
| 0 | 成功 | | 2003 | 重复报名 |
| 1001 | 未登录/Token失效 | | 2004 | 开标前/未解密不可下载 |

## 11. 包结构（Spring Boot 3）

```text
com.guandian.bidding
├─ common / config / security
└─ module
   ├─ auth        认证/用户/角色切换/角色申请/邀请招标人
   ├─ tender      招标项目(状态机)/公告/搜索
   ├─ bid         报名/缴费(payment)/投标文件(加解密)
   ├─ evaluation  抽专家/评审项/评分/谈判/二轮报价/评标报告
   ├─ award       中标公示/代理费/通知书/归档
   ├─ content     文章(指南/法规/案例/下载/动态)/投诉建议
   └─ notify      消息通知
```
> 依赖：Spring Boot 3.x、Security+JWT、MyBatis-Plus、MySQL、Redis、Knife4j、MinIO。
